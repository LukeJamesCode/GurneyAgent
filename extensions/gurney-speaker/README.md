# gurney-speaker

Handmade smart-speaker front-end for Gurney. The extension hosts a WebSocket server that ESP32-S3 "Gurney puck" devices connect to: they stream microphone audio in (raw 16 kHz mono PCM after on-device wake-word detection) and play TTS replies back (OGG/Opus). Display state (`idle | listening | thinking | speaking | muted`) and volume/mute updates ride the same connection.

It depends on **gurney-voice** for whisper.cpp STT and Piper TTS â€” no duplicated audio plumbing.

> **Status:** server side fully working in v0.1 â€” WS server, session machine, STT, LLM dispatch, and TTS are all wired and unit-tested. Firmware is not started yet.
>
> **v0.1 dispatch is LLM-only.** The device chats with `host.llm` directly (no tools, no memory, no conversation history). Tool calls (calendar, reminders, etc.) won't fire from the device until a future release adds a small orchestrator host helper â€” but the protocol and session don't need to change when that lands; only `pipeline.ts` does.

- `device_shared_secret` â€” generated by setup; flash into the firmware NVS image.
- `display_style` â€” `minimal` or `orb`; pushed to the device in the welcome frame.
