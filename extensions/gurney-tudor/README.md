# gurney-tudor

A guided-learning studio for Gurney — NotebookLM-style, but built for a
CPU-only, qwen-native box.

Give it a topic and Tudor **compiles a whole interactive course up front** —
modules → lessons → bite-size slides, with checkpoints, quizzes, flashcard
review, and a mastery map. The slow, expensive thinking happens once, as a
background job behind a live progress view; after that, **stepping through the
course is instant** because nothing calls a model in real time. That's how the
40–60s local-inference latency is hidden from the learner.

## What you get

- **Learn tab** in the Gurney web panel (`gurney-frontend`). The tab only
  appears when this extension is enabled.
- **`/learn <topic>`** Telegram command to kick off a build from chat; open the
  panel to watch it compile and start learning.

## How a course is built

1. **Outline** — one quick model call produces the syllabus (title, modules,
   lesson titles). The skeleton shows up almost immediately.
2. **Lessons** — each lesson is generated by its own small, focused call, one at
   a time. Lessons unlock as they finish, so you can start lesson 1 while the
   rest keep compiling.

To stay reliable on a 0.8B/2B model, Tudor does **not** ask for nested JSON. It
uses a flat, line-tagged format (`TITLE:` / `MODULE:` / `- lesson`, and
`SEGMENT: kind` / `QUIZ:`) that's trivial for a small model to emit and
bulletproof to parse — and every parse degrades gracefully (a lesson with no
recognised markers still becomes one readable slide).

## Learning features

- **Step-through slides** — one idea at a time, with keyboard navigation.
- **Typed segments** — explain / example / analogy / key points / checkpoint /
  warning, each themed so a lesson has visual rhythm.
- **Checkpoint quizzes** — instant feedback and an explanation.
- **Explain simpler / go deeper** — an on-demand rewrite of any slide. The first
  request runs a local model call; the result is cached on the slide so a repeat
  is instant and costs nothing.
- **Flashcard Review** — turns a course's quizzes and key points into an
  active-recall deck. Zero extra model cost.
- **Mastery map** — per-lesson progress plus a self-rated confidence that fills
  a mastery bar, so you can see what to revisit.

## Settings (`gurney config gurney-tudor`)

| Setting             | Default    | Notes                                                                                   |
| ------------------- | ---------- | --------------------------------------------------------------------------------------- |
| `default_generator` | `local`    | `local` builds on your Ollama/qwen model (free). `codex` uses `gurney-codex` if authed. |
| `default_depth`     | `standard` | `quick` / `standard` / `deep` — how many modules and lessons a new course aims for.     |

Codex is opt-in and best-effort: if it's unavailable or its daily budget is
spent mid-build, Tudor automatically finishes the course on the local model.

## Data

Everything lives in the shared SQLite DB under `tudor_*` tables (see
`migrations/0001_tudor.sql`): courses, modules, lessons, segments, quizzes,
per-lesson progress, and a generation-job row per course. Uninstalling with
`--purge` drops them.
